# Data Architecture Advisor

Data Architecture Advisor is a browser-based assessment tool for enterprise data architecture maturity. It helps organisations evaluate current capability, identify risk, and generate a practical consulting-style roadmap across open source tooling, AWS managed services, governance, data quality, security, cost, operating model, lifecycle management, and AI readiness.

## Features

- Multi-step enterprise assessment questionnaire across 24 sections.
- Maturity scoring from 1 to 5 across 19 architecture dimensions.
- Recommendation engine for open source heavy, hybrid, and AWS managed modes.
- Consulting-grade in-browser report with executive summary, maturity classification, risk heatmap, radar chart, bar chart, architecture diagram, priority matrix, roadmap timeline, and cost vs complexity comparison.
- Tool recommendations for DataHub, Great Expectations, MetaWorks, Amazon S3, Amazon Redshift, AWS Glue, AWS Lambda, Step Functions, Amazon Kinesis, Apache Airflow, dbt, Apache Superset or Metabase, Prometheus and Grafana, ClickHouse, and Airbyte.
- Local browser persistence with `localStorage`.
- Restart assessment and print/export report actions.
- Responsive design and print-friendly report layout.
- GitHub Pages deployment workflow.

## Deployment

The repository includes a GitHub Actions workflow at `.github/workflows/deploy.yml`.

1. Push changes to `main`.
2. In the GitHub repository settings, set Pages source to GitHub Actions if it is not already selected.
3. The workflow builds the Vite app and deploys the `dist` artifact to GitHub Pages.

Manual deployment is also available:

```bash
npm run deploy
```

## Assessment Methodology

The assessment scores questionnaire responses from 1.0 to 5.0 and aggregates them into architecture maturity dimensions such as ingestion, storage, transformation, catalogue and metadata, governance, data quality, observability, security and compliance, cost efficiency, lifecycle and retention, ownership, data contracts, CI/CD, environment management, lineage, SLAs and freshness, ML/AI readiness, and cloud portability.

Overall maturity is classified as:

- 1.0 to 1.9: Ad hoc
- 2.0 to 2.9: Developing
- 3.0 to 3.9: Managed
- 4.0 to 4.5: Advanced
- 4.6 to 5.0: Optimised

Recommendations are generated by comparing current maturity against a target architecture posture and mapping gaps to practical actions, open source tooling, AWS managed services, governance controls, operating model improvements, and a three-phase roadmap.
